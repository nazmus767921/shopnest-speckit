## Requirements
### Requirement: Global Theme Configuration
The system SHALL allow merchants to configure global theme settings, including primary color, secondary color, background color, text color, heading font, body font, and border radius.

#### Scenario: Merchant updates theme colors
- **WHEN** merchant saves new theme colors in the dashboard
- **THEN** the store's storefront immediately updates to use the new colors via CSS variables injected at the layout level

#### Scenario: Merchant updates typography settings
- **WHEN** merchant selects a new heading font "Cormorant Garamond" and body font "Lato" in the dashboard and saves
- **THEN** the store's storefront renders headings with Cormorant Garamond and body text with Lato via CSS variables `--font-heading` and `--font-body`

### Requirement: Theme Settings Injection
The storefront layout SHALL inject the merchant's saved `theme_settings` as CSS variables onto the root HTML element. The layout SHALL apply all curated font CSS variable classes from the font registry so that font custom properties are available to all descendant elements. Typography CSS variables (`--font-heading`, `--font-body`) SHALL be generated by `getThemeVariables` when typography settings are present.

#### Scenario: Storefront renders with custom theme
- **WHEN** a customer visits a store with configured theme settings
- **THEN** the Next.js layout renders `<html style={{ '--color-primary': '...', '--font-heading': '...' }}>` based on the saved settings

#### Scenario: Storefront renders with default fonts when no typography set
- **WHEN** a customer visits a store that has not configured typography settings
- **THEN** the storefront uses the default font stack (sans-serif) with no `--font-heading` or `--font-body` overrides
