## Why

The storefront templates (General and Fashion) are currently ignoring several dynamic settings saved in the dashboard, particularly typography, social links, and some section-specific visual settings. This creates a disconnected user experience where merchants change settings in the builder but see no effect on their live storefront. We need to correctly wire these up so customization works end-to-end.

## What Changes

- **Typography Wire-up**: Remove hardcoded `--font-display` and `--font-sans` overrides in both `general` and `fashion` stylesheets, and map them to the global `--font-heading` and `--font-body` variables generated by the dashboard.
- **Component Font Classes**: Remove hardcoded Tailwind font classes (`font-sans`, `font-serif`) from `FullBleedHero.tsx`, `BrandStory.tsx`, `FaqSection.tsx`, etc., so they inherit the dynamic font variables.
- **Social Links Expansion**: Update `FashionFooter.tsx` to render WhatsApp and TikTok social icons (currently it only renders Facebook and Instagram).
- **WhatsApp URL Fix**: Fix the URL parsing in `GeneralFooter.tsx` and `FashionFooter.tsx` so that pasting full URLs (like `https://api.whatsapp.com/...`) doesn't break due to aggressive non-numeric character stripping.
- **Hero Overlay Opacity**: Implement the `overlayOpacity` setting in `FullBleedHero.tsx` so that background images are actually darkened, ensuring text readability.
- **Category Mosaic Styling**: Update `CategoryMosaic.tsx` to respect dynamic theme colors instead of hardcoding black overlays and white text on hover.

## Capabilities

### New Capabilities
- None.

### Modified Capabilities
- `storefront-builder`: Updating the template engine to correctly apply typography, footer social links, and section-level styling constraints to the rendering pipeline.
