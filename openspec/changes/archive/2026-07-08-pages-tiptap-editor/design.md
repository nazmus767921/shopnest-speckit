# Technical Design: Pages Tiptap Editor

This document outlines the technical design for replacing the plain HTML `<textarea>` on standard pages with a custom, premium WYSIWYG editor powered by Tiptap.

## Context
Merchants edit custom pages like "About Us" and "Shipping Policy" in the dashboard. The current interface has a simple `<textarea>` which is cumbersome. We will implement Tiptap, a headless rich text editor framework, and style it to match the ShopNest design system.

## Goals / Non-Goals

### Goals
- Integrate `@tiptap/react` and essential extensions (`@tiptap/starter-kit`, `@tiptap/extension-link`, and `@tiptap/extension-placeholder`).
- Style the editor toolbar and content area using ShopNest `DESIGN.md` rules (no shadows, pill buttons only, tailored colors, two-track polarity, and Inter typography).
- Define custom, responsive CSS rules for standard tags inside `.prose` and `.ProseMirror` in `globals.css` to render rich text headers, lists, blockquotes, and links properly.
- Ensure the editor integrates cleanly with `@tanstack/react-form` in `PageForm`.

### Non-Goals
- Supporting image/video uploads or media embeds inside standard pages.
- Changing database schema or queries (multi-tenant boundaries remain unchanged).

## Decisions

### Decision 1: Option A (Custom CSS rules for `.prose` and `.ProseMirror`)
Instead of installing `@tailwindcss/typography` which introduces a heavy dependency and Tailwind configuration friction under Tailwind v4, we will write explicit, highly tailored CSS rules directly inside `globals.css` for both the storefront pages (`.prose`) and the Tiptap editor workspace (`.ProseMirror`).
- **Rationale**: Keeps the bundle light, fits perfectly with design tokens in `DESIGN.md`, and is easy to customize.
- **Alternatives Considered**: Option B (Tailwind Typography plugin), which was rejected because Tailwind v4 handles plugins differently and custom styling matches our precise design tokens much better.

### Decision 2: Toolbar & Dialog Styling Compliance with DESIGN.md
In accordance with ShopNest non-negotiable invariants, the editor UI will have:
- No shadow classes (e.g., no `shadow-*`).
- Pill buttons only (`rounded-full`) for all interactive actions (e.g. Save, Cancel, and any toolbar toggle icons if styled as buttons).
- Background colors matching the light-cream track (`bg-canvas-light`, borders using `border-hairline-light`, text using `text-ink`).

### Decision 3: Next.js 16 Caching Compatibility
- Storefront page fetching uses `"use cache"` with `cacheLife("hours")` and dynamic cache tags (`merchant-${merchantId}-pages` and `page-${merchantId}-${slug}`).
- The existing server actions `createPageAction` and `updatePageAction` already invoke `revalidateTag(..., "max")` and `revalidatePath(...)` to invalidate the cache. We will leave these as is since they are completely compatible with the new rich text input format.

### Decision 4: Normalize Empty Output
To avoid storing redundant empty tags (like `<p></p>` or `<p><br></p>`) in the database when the merchant clears the text editor:
- The `RichTextEditor` component will check if `editor.isEmpty` is true or if the content is just empty whitespace, and will call `onChange("")` with an empty string.

### Decision 5: Safe HTML Paste Formatting
Tiptap's schema constraints will naturally strip out layout-breaking CSS inline styling, custom fonts, and illegal tags (like `<script>`) when text is pasted from sources like Google Docs or external websites. No external sanitizer is needed inside the editor.

### Decision 6: Link Normalization
The Link extension will be configured to automatically parse pasted URLs (`linkOnPaste: true`) and we will ensure the Link Insertion Bubble normalizes external domains by adding `https://` if no protocol is specified.

## Risks / Trade-offs

- **[Risk] Hydration Warnings/Flicker**: Tiptap is a browser-only library. Running it during server-side prerendering can trigger hydration errors.
  - *Mitigation*: Dynamically import `RichTextEditor` with `ssr: false` in `page-form.tsx` or wrap the instantiation in a client-only mounting state.

## Test Strategy
